**エージェント向け（機械可読）**: [docs/DESIGN.md](docs/DESIGN.md) は [google-labs-code/design.md](https://github.com/google-labs-code/design.md) 形式（YAML フロントマター＋本文）。`npm run design:lint` で仕様上の検証ができる。数値の食い違いは **実装**（`global.css` / `tailwind.config.mjs`）と **この文書** を優先し、`docs/DESIGN.md` の `mun-border` 等、lint 用に近似hexへ寄せた箇所は [docs/DESIGN.md の Colors 節](docs/DESIGN.md) の注記に従う。

## ブランド概要

- **性質**: 模擬国連・国際法・国際政治に関する編集型メディア。静的な読み物が中心。
- **トーン**: 落ち着いた学術・エディトリアル。装飾より可読性と階層。
- **リブランドの方向性**: 従来の「紫〜青のグラデーション散在」を、**深いネイビー基調 + 単一アクセント系**に整理し、カテゴリ色は意味が分かる範囲で抑制したパレットに統一する。

上記の外に、**思想的正本**は [docs/PHILOSOPHY.md](docs/PHILOSOPHY.md)。迷ったら先にそこに立ち返り、**見た目の判断**は下表と以降の章に従う。

## PHILOSOPHY との対応

| 哲学の柱（要約） | デザイン上の含意 |
|------------------|------------------|
| 会期の**外側**の学び（*Beyond the Gavel*） | **議場の熱量・ダッシュボード**より、**図書室・良い新聞の解説欄**に近い落ち着き。装飾で煽らない。 |
| **根拠と論点が追える**文章 | タイポの**階層＝主張の階層**（H2 大論点、H3 下位）。日本語見出しが主役、英語副題は補助。 |
| 編集型メディア・学術的厳しさ | **色数の抑制**、カテゴリ4色の意味づき、**メタデータ**（日付・著者・カテゴリ）を読み物と並べて信頼を示す。 |
| 公平なアクセス | **可読性**（行長・行間・コントラスト・フォーカス）を優先。注意を釣るマーケ色より、落ち着いた面。 |

**望ましいパターン**: 静かなベース、明確な枠（`border-mun-border`）、CTA・リンクに限定したアクセント、必要最小限のモーション。長文（学問的探究）では、将来 **目次・出典まわり**を追加する場合も、同じトークンとコンポーネントに揃える。

**避ける物語・見た目**: ゲーム的バッジの多用、常時アニメ背景、ガベル等の**比喩の過剰イラスト化**（英語副題のタイポで足りる箇所が多い）、週刊誌的な煽りトーン。

## タイポグラフィ

- **フォントスタック**: `Inter`, `Noto Sans JP`, `system-ui`, `sans-serif`
- **見出し（UI）**: `font-bold` 〜 `font-black`。ヒーローは `hero-gradient-text` ユーティリティ（アクセントグラデーション）。
- **記事本文**: `.markdown-content` 内。`font-noto`、行間 1.75〜1.8、適度な `letter-spacing`。

## スペーシング・シェイプ

- **横枠（サイト全体）**: `layout-shell` = `max-w-7xl mx-auto` + レスポンシブ横パディング（`px-4 sm:px-6 lg:px-8`）。**長文1カラム**は `layout-narrow` または `layout-article`（`max-w-3xl`）。**幅広ブロック**は `layout-wide`（`max-w-5xl`）や `max-w-6xl` 相当を `layout-shell` 内に配置。ヘッダー・フッター・本文で横位置を揃える。
- **トップFVのテキスト列**: `layout-hero-inner`（`max-w-3xl` + ガター）。
- **セクション縦余白**: `py-16 md:py-20 lg:py-24` 系を目安。`section-pad` ユーティリティも可。
- **角丸**: カード `rounded-xl`、ボタン `rounded-lg`、ピル型 CTA は `rounded-full` を限定利用。
- **ヘッダー下オフセット**: 固定ヘッダー下のページ本体先頭に `pt-16`（`.page-main` 相当）。

## コンポーネント指針

- **Button**: プライマリは `mun-accent` 基調。セカンダリは境界線のみのゴースト。
- **Card**: `card-bg`（半透明サーフェス + 軽い blur + `mun-border`）。
- **Badge / Message**: ダーク UI 上で読める **半透明背景 + 明るい前景**（ライトグレー地のままにしない）。

## Markdown 記事ルール

- 記事内 **H1 は使わない**（ページテンプレートが H1 を持つ）。本文は **H2 から**。
- 長文は **H2 → H3** で階層化。飛び級を避ける。
- 装飾記号（例: セクション見出し脇の記号）は **使わない**。区切りはタイポと余白で表現する。
- 画像は `alt` 必須。出典・ライセンスがあればキャプションか直下の段落に記載。

## アクセシビリティ

- スキップリンク: `BaseLayout` 内 `#main-content` へ。
- **ドキュメント内に `main` は1つだけ**（`BaseLayout` が担当）。ページ側は `div.page-main` 等でラップ。
- フォーカス: 可視のアウトラインを消さない（`outline` は `mun-focus` に合わせる）。

## 計測・サードパーティ

- **GTM を正**。タグの追加・削除は GTM 管理画面で行い、ページに計測タグを重ねない。
- フォーム（Brevo）の見た目はサイトトークン（フォーカスリング・アクセント）に合わせる。

## 禁止・非推奨

- セクションタイトル周辺の **記号による装飾**（§、大きな引用符、ドットパターン背景、装飾用アイコン円の多用など。読み上げ・視覚ノイズになりやすい）。
- **グラデーション見出しの乱用**。ヒーローや強調の一点以外は、実色の `text-mun-text` と字重で階層をつける。
- Tailwind の **動的クラス名連結**（`text-${color}-400` 等）。JIT が生成しないため **マップで明示**する。
- `main` のネスト。

## ページ構成の考え方

- **トップのヒーロー**: 全幅背景画像は `public/images/hero-index.png`（`.index-hero` in `global.css`）。**差し替え**は同パスに同名で置き換える。雰囲気は図書室・エディトリアル（派手な放射グラデや国旗モチーフの多用は避ける）。
- **上記以外のページ先頭**は `page-hero`（下線付きの落ち着いた帯）でタイトルとリードをまとめる。中央配置の装飾ヒーロー＋放射グラデーションは避ける。
- **セクション**は `border-t border-mun-border` で区切り、見出しは左寄せ・`section-heading` 程度のサイズに抑える。
- **リンクと CTA**は文言と下線・コントラストで示し、矢印アイコンや絵文字は必要最小限にする。
